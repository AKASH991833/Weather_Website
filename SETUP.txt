================================================================================
                         WEATHERNOW - SETUP GUIDE
================================================================================

PREREQUISITES
-------------
- Modern web browser (Chrome, Firefox, Safari, Edge)
- OpenWeatherMap API account (free tier available)
- Local web server (for development) or static hosting


STEP 1: GET YOUR API KEY
------------------------
1. Visit https://openweathermap.org/api
2. Click "Sign Up" and create a free account
3. Navigate to API Keys section in your dashboard
4. Copy your API key (looks like: abc123def456...)
5. Note: Free tier includes:
   - 60 calls/minute
   - 1,000,000 calls/month
   - Current weather, 5-day forecast, geocoding


STEP 2: CONFIGURE API KEY
-------------------------
1. Navigate to the js/ folder
2. Copy config.js.example to config.js:
   
   cp js/config.js.example js/config.js
   
   (On Windows PowerShell: Copy-Item js/config.js.example js/config.js)

3. Open js/config.js in a text editor
4. Replace 'YOUR_API_KEY_HERE' with your actual API key:
   
   API_KEY: 'your_actual_api_key_here',

5. Save the file
6. IMPORTANT: Never commit config.js to version control!


STEP 3: RUN LOCALLY
-------------------
Option A - Using Python (recommended for quick testing):
   
   # Python 3
   python -m http.server 8000
   
   # Python 2
   python -m SimpleHTTPServer 8000
   
   Then open: http://localhost:8000

Option B - Using Node.js (http-server):
   
   npm install -g http-server
   http-server -p 8000
   
   Then open: http://localhost:8000

Option C - Using VS Code Live Server:
   
   1. Install "Live Server" extension
   2. Right-click index.html
   3. Select "Open with Live Server"

Option D - Using PHP:
   
   php -S localhost:8000


STEP 4: VERIFY SETUP
--------------------
1. Open the application in your browser
2. Search for a city (e.g., "London")
3. Verify weather data displays correctly
4. Check the map loads properly
5. Test the 7-day forecast display
6. Try the "Use my location" button


TROUBLESHOOTING
---------------
Problem: "Invalid API key" error
Solution: 
  - Double-check your API key in config.js
  - Ensure no extra spaces or quotes
  - Wait 10-15 minutes after creating new API key

Problem: Map not loading
Solution:
  - Check browser console for errors
  - Ensure Leaflet JS/CSS are loading
  - Verify internet connection

Problem: Location permission denied
Solution:
  - Click the lock icon in address bar
  - Allow location permissions
  - Refresh the page

Problem: Search not returning results
Solution:
  - Ensure query is at least 2 characters
  - Try more specific city names
  - Check API rate limits

                       DEPLOYMENT INSTRUCTIONS
================================================================================

DEPLOY TO NETLIFY
-----------------
1. Create account at https://netlify.com
2. Install Netlify CLI (optional):
   
   npm install -g netlify-cli

3. Drag and drop your project folder to Netlify dashboard
   OR use CLI:
   
   netlify deploy --prod

4. Set environment variables in Netlify dashboard:
   - Not needed for this setup (config.js handles it)

5. Your site will be live at: your-site.netlify.app

Custom Domain (optional):
  - Go to Domain Settings
  - Add custom domain
  - Update DNS records as instructed


DEPLOY TO VERCEL
----------------
1. Create account at https://vercel.com
2. Install Vercel CLI:
   
   npm install -g vercel

3. Navigate to project folder and run:
   
   vercel

4. Follow prompts:
   - Set up and deploy? Y
   - Which scope? (select your account)
   - Link to existing project? N
   - Project name? (enter name)
   - Directory? ./
   - Override settings? N

5. Your site will be live at: your-project.vercel.app

Environment Variables (if needed):
  - Go to Project Settings > Environment Variables
  - Add VITE_API_KEY or similar


DEPLOY TO GITHUB PAGES
----------------------
1. Create a GitHub repository
2. Push your code:
   
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/username/repo.git
   git push -u origin main

3. Go to repository Settings > Pages
4. Select source: Deploy from branch > main > / (root)
5. Save
6. Your site will be live at: username.github.io/repo


DEPLOY TO CLOUDFLARE PAGES
--------------------------
1. Create account at https://pages.cloudflare.com
2. Connect your GitHub repository
3. Configure build settings:
   - Build command: (leave empty - no build needed)
   - Build output directory: /
4. Deploy
5. Your site will be live at: your-project.pages.dev


================================================================================
                         SECURITY BEST PRACTICES
================================================================================

API KEY SECURITY
----------------
1. NEVER commit config.js to version control
   - Already in .gitignore
   - Double-check before committing

2. For production, consider:
   - Using a backend proxy to hide API key
   - Implementing API key rotation
   - Setting up API key restrictions in OpenWeather dashboard

3. Monitor API usage:
   - Check OpenWeather dashboard regularly
   - Set up usage alerts
   - Implement client-side rate limiting


RATE LIMITING (Client-side)
---------------------------
The app includes:
- Debounced search (300ms delay)
- Response caching (10 minutes)
- Cache cleanup (every 5 minutes)

For additional protection, add to main.js:
   
   const requestQueue = [];
   const MAX_REQUESTS_PER_MINUTE = 50;
   
   function queueRequest(fn) {
     // Implement request queuing
   }


INPUT VALIDATION
----------------
- Search queries validated (min 2 characters)
- Coordinates validated before API calls
- HTML sanitization for user inputs


================================================================================
                         PERFORMANCE OPTIMIZATION
================================================================================

LAZY LOADING
------------
- Map loads only when visible (Intersection Observer)
- Weather icons use loading="lazy"
- Modules use ES6 dynamic imports


CACHING STRATEGY
----------------
- API responses cached for 10 minutes
- Location preferences saved to localStorage
- Search results cached by query


BUNDLE OPTIMIZATION
-------------------
For production, consider:
- Minify CSS/JS files
- Use CSS purging tools
- Implement service worker for offline support
- Add compression (gzip/brotli)


CDN USAGE
---------
External resources loaded from CDN:
- Leaflet CSS/JS (unpkg.com)
- Weather icons (openweathermap.org)
- Map tiles (openstreetmap.org)
